# Aircraft IoT Threat Modeling Matrix

## Learning Objectives
- Apply STRIDE threat modeling methodology
- Identify threats to aircraft IoT systems
- Prioritize risks based on asset impact

## How to Run
1. Open `index.html` in a browser
2. Review threat categories
3. Add new threats and view the list

## STRIDE Categories
- **Spoofing**: Impersonating another device/entity
- **Tampering**: Modifying data or code
- **Repudiation**: Ability to deny actions
- **Information Disclosure**: Exposing sensitive data
- **Denial of Service**: Preventing legitimate use
- **Elevation of Privilege**: Gaining unauthorized access

## Exercise
Identify 2-3 threats for each aircraft IoT component.
