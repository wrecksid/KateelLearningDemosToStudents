# Queue Design and Management in a Bank Branch

## Overview
This project demonstrates synthetic data generation and queue analysis in a bank branch setting using AI/ML techniques.

- `synthetic_data_generator.py` : Generates synthetic customer queue dataset using Faker (locale: en_IN).
- `queue_management_demo.py` : Uses synthetic data to analyze queue metrics, produce plots, and offer management insights.
- `requirements.txt` : Python dependencies.
- `setup_venv.sh` & `setup_venv.bat` : Scripts to create python virtual environment on Linux and Windows.
- `syntheticdata.csv` : Example synthetic data output (generated by the first script).
- Jupyter notebooks for both programs enable interactive exploration.

## How to Run

1. Set up Python environment:
   - Linux: `bash setup_venv.sh`
   - Windows: `setup_venv.bat`
   - Activate environment and install dependencies:
     ```
     pip install -r requirements.txt
     ```

2. Generate synthetic data:
python synthetic_data_generator.py --num_records 10000 --output_file syntheticdata.csv

3. Run Queue Management Demo:
python queue_management_demo.py


Alternatively, open and run the Jupyter notebooks (`SyntheticDataGenerator.ipynb`, `QueueManagementDemo.ipynb`) from the `BankQ` folder in JupyterLab or Google Colab.

## How Banks Can Use This Demo
- Plan staff allocation based on analyzed wait times and customer arrival patterns.
- Optimize queue management by simulating various customer inflow and transaction scenarios.
- Enhance customer satisfaction through data-driven scheduling during peak hours.
- Evaluate impact of transaction types on service times to prioritize critical customers.
- Use as a training tool for branch managers and operations analysts to understand queue dynamics quantitatively.
