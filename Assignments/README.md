# Assignments

**Repository:** [KateelLearningDemosToStudents](https://github.com/VinayaSharada/KateelLearningDemosToStudents)
**Author:** Professor Vinaya Sathyanarayana

This folder converts repository demos into course-ready assignment scaffolds.
Each assignment folder maps to one or more demos and provides a structured problem
statement with learning goals, required outputs, and extension tasks.

---

## Intended Use

- Graded lab exercises
- Weekly homework sets
- Mini project briefs
- Mid-term case studies
- Final project preparation

These scaffolds are designed so a student can open the corresponding demo, run it,
understand the output, and then apply or extend the technique for the assignment.

## Assignment Folder Structure

Each assignment folder should contain:

- **README.md** — learning goals, problem statement, required outputs, extension tasks, evaluation rubric
- Reference to the prerequisite demo(s) with setup instructions
- Any additional data files or starter notebooks specific to the assignment

---

## For Faculty: Adding New Assignments

1. Create a new folder named `Session<NN>_<Topic>`.
2. Copy the structure from an existing assignment folder.
3. Link it to one or more demos from [DomainUseCaseDemos](../DomainUseCaseDemos/) or [TechUseCaseDemos](../TechUseCaseDemos/).
4. Define clear deliverables and a rubric.
5. Submit a pull request — see [CONTRIBUTING.md](../CONTRIBUTING.md).
